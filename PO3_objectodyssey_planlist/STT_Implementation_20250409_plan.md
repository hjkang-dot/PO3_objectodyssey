# STT 기능 구현 및 JSON 파싱 오류 수정 계획 (2025-04-09)

## 배경 및 목표
현재 서비스에서 음성 입력(STT) 기능을 사용하려 할 때, 백엔드에 해당 엔드포인트(`/stt`)가 없어 404 오류가 발생하고 있습니다. 또한, 프론트엔드에서 이 오류 응답(HTML 형태)을 JSON으로 파싱하려다 보니 `SyntaxError`가 발생하고 있습니다. 이 문제를 해결하여 원활한 음성 입력 환경을 제공하는 것이 목표입니다.

## 주요 수정 사항

### 1. 백엔드: `/stt` 엔드포인트 추가
- **파일**: [odyssey_audio.py](file:///c:/project_3rd/PO3_objectodyssey/unified_app/odyssey_audio.py)
- **내용**: 
    - OpenAI Whisper 모델을 사용하여 업로드된 오디오 파일을 텍스트로 변환하는 기능을 구현합니다.
    - `/stt` 엔드포인트를 추가하여 프론트엔드의 요청을 처리합니다.

### 2. 프론트엔드: 오류 핸들링 강화
- **파일**: [index.html](file:///c:/project_3rd/PO3_objectodyssey/unified_app/static/index.html)
- **내용**: 
    - `stopRecordingAndSend` 함수에서 응답 상태를 확인하고, JSON 파싱 실패 시 적절한 에러 메시지를 출력하도록 수정합니다.

## 서비스 플로우 (수정됨)
1. 사용자가 페이지 접속 (`/`)
2. 성별 선택
3. 장난감 사진 촬영 및 정보 입력 (**신규: 마이크 버튼으로 음성 입력 지원**)
    - 사용자가 마이크 버튼 클릭 시 녹음 시작
    - 버튼을 떼면 `/stt` 호출
    - 백엔드에서 OpenAI Whisper로 텍스트 변환 후 반환
    - 입력창에 변환된 텍스트 자동 삽입
4. `/extract` 호출 (누끼 따기 및 설명 생성)
5. `/create-art` 호출 (캐릭터 이미지 생성)
6. 동화 톤 및 목소리 선택
7. `/generate-book` 호출 (스토리 및 음성 생성)
8. 동화책 읽기 (클라이언트측 재생)
9. 보관함 저장 (`/save-story`)

## 검증 계획
- 브라우저 실행 후 주인공 정보 입력창 옆 마이크 버튼 클릭하여 음성 인식 확인.
- 오디오 파일 전송 시 백엔드에서 정상적으로 텍스트로 변환되는지 로그 확인.
