# 이미지 및 TTS 로딩 문제 해결 계획

사용자가 보관함에서 동화를 열었을 때 이미지와 TTS가 동작하지 않는 문제를 해결하기 위한 계획입니다.

## 1. 개요 및 원인 분석
- **TTS 미동작**: `renderSavedStory`에서 `renderBook`을 먼저 호출하고 `showScreen`을 나중에 호출합니다. `renderBook` 내부에서 TTS 재생을 시도할 때 아직 화면이 `screen-library` 상태이므로, "스토리 화면에서만 재생"이라는 안전 로직에 걸려 재생이 차단되었습니다.
- **이미지 미표시**: 
    - `adjustPath` 함수가 보관함의 각기 다른 부분에서 서로 다르게 구현되어 있어, `saved_stories/` 경로를 `stories/`로 보정해주는 로직이 일부 누락되었습니다.
    - 오디오 경로(`page_audios`)에 대해서는 경로 보정 로직이 적용되지 않았습니다.
    - 한글 파일명이 포함된 경로의 경우 브라우저 특성에 따라 인코딩 문제가 발생할 수 있습니다.

## 2. 상세 수정 계획

### [Component] Frontend (static/index.html)
#### [MODIFY] [static/index.html](file:///c:/project_3rd/PO3_objectodyssey/unified_app/static/index.html)
- **공용 경로 보정 함수 추가**: `safeAssetPath`라는 이름의 강력한 경로 보정 함수를 전역적으로 정의하여 사용합니다. (역슬래시 보정, `saved_stories/` -> `stories/` 변환, 슬래시 중복 방지 포함)
- **TTS 자동 재생 타이밍 수정**: `renderSavedStory`에서 `showScreen('screen-story')`를 `renderBook(mockData)`보다 **먼저** 호출하도록 순서를 변경합니다.
- **오디오 경로 보정 적용**: `page_audios` 배열의 각 항목에도 `safeAssetPath`를 적용합니다.
- **인코딩 처리**: 경로 보정 시 필요한 경우 `encodeURI`를 고려하되, 기본적으로 올바른 URL 형식을 갖추도록 합니다. 중복된 `adjustPath` 로직들을 하나로 통합합니다.

## 3. 검증 계획
### 수동 테스트
1. 보관함 진입 후 이전에 생성된 동화를 클릭합니다.
2. 화면이 바뀌자마자 첫 페이지의 TTS가 자동으로 들리는지 확인합니다.
3. 각 페이지의 이미지가 엑스박스 없이 정상적으로 표시되는지 확인합니다.
4. (이미 생성된 동화가 있는 경우) `saved_stories/` 경로로 저장된 옛날 데이터도 정상적으로 로드되는지 확인합니다.

## 4. 수정 완료 (Implementation Result) - 2026-04-09
- **경로 보정 전역화**: `safeAssetPath` 함수를 구현하여 `saved_stories/` -> `stories/` 변환 및 슬래시 보정을 통일했습니다.
- **이미지 및 오디오 해결**: 보관함 로딩 시 이미지와 오디오 모든 경로에 `safeAssetPath`를 적용하여 404 오류를 방지했습니다.
- **TTS 자동 재생 수정**: 보관함에서 동화 선택 시 화면 전환(`showScreen`)을 먼저 수행하도록 하여 TTS 재생 조건(현재 화면 체크)을 만족시켰습니다.
