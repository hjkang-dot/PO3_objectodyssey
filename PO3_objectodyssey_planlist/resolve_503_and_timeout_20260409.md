# 작업 계획 및 기록: 503 에러 해결 및 서버 안정성 강화 (2026-04-09)

## 1. 개요
동화 생성 과정에서 발생하는 503 Service Unavailable 에러와 타임아웃 문제를 해결하기 위해 서버와 클라이언트의 설정을 보완합니다.

## 2. 수정 계획 및 이유

| 파일 | 수정 내용 | 이유 |
| :--- | :--- | :--- |
| `gemini_service.py` | `HttpRetryOptions` 및 `timeout` 추가 | API 일시 오류 시 자동 재시도로 성공률 제고 |
| `storybook_core.py` | `max_workers` 축소 (6 -> 3) | 동시 API 요청 수를 줄여 부하 및 503 발생 억제 |
| `main.py` | `timeout_keep_alive` 상향 (60초) | 긴 작업 시간 동안 연결 유지 보장 |
| `index.html` | 503 에러 전용 안내 메시지 추가 | 사용자에게 상황을 알리고 재시도 유도 |

## 3. 수정 완료 내역

### [2026-04-09 13:00] 심화 안정화 작업 완료
- **`storybook_core.py`**: 이미지 생성을 완전 직렬(Sequential) 방식으로 전환하고, 생성 간 2초의 `time.sleep`을 추가하여 API 과부하 가능성을 원천 차단했습니다.
- **`gemini_service.py`**: 재시도 횟수를 8회로 늘리고, 첫 대기 시간을 5초로 상향하여 Google 서버 복구 시간을 충분히 확보했습니다.
- **`index.html`**: `startStoryGen` 함수에 프런트엔드 차원의 자동 재시도 로직(최대 2회)을 추가하고, 사용자에게 1~2분 정도 소요됨을 미리 안내하도록 UX를 개선했습니다.

## 4. 전체 서비스 플로우 (최종 업데이트)
1. 사용자: 이미지 업로드 -> 객체 추출 (`/extract`)
2. 사용자: 캐릭터 정보/톤 선택 -> 캐릭터 아트 생성 (`/create-art`)
3. **사용자: 동화 마법 부리기 -> 스토리/이미지 생성 (`/generate-book`)**
   - *변경점*: 이미지를 **하나씩 순차적으로** 생성하며, 각 단계 간 지연 시간을 두어 안정성을 극대화함.
   - *변경점*: 브라우저에서 503 에러 발생 시 **자동으로 다시 요청**을 보내는 2중 방어막 구축.
4. 사용자: 동화 감상 및 보관함 저장 (`/save-story`)
